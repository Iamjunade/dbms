Project Documentation

Comprehensive overview of the ISP Billing Management System micro-project.

1. Abstract

This project demonstrates a modern, web-based Database Management System (DBMS) for an Internet Service Provider (ISP). It provides a luxury user interface for managing customer records, billing plans, and invoices. The system features a fully interactive CRUD interface for customer management and a pseudo-SQL console for direct database interaction, simulating real-world database operations. The backend is powered by Supabase, a scalable cloud database, ensuring data persistence and reliability.

2. Key Features

  • Customer Management: Full CRUD (Create, Read, Update, Delete) functionality for customer records.
  • Dynamic Billing Plans: Plans are fetched dynamically from the database, allowing for easy updates.
  • Interactive DB Console: Execute simplified SQL queries (`SELECT`, `DELETE`) to directly manipulate the database.
  • Relational Schema Visualization: A dedicated view to display the normalized database architecture (3NF).
  • Persistent Data Storage: Utilizes Supabase (PostgreSQL) for live, persistent data storage.
  • Modern Frontend: Built with Angular using Signals for state management, providing a reactive and performant UI.

4. Database Schema Notes

The database schema is designed following the principles of Third Normal Form (3NF) to reduce data redundancy and improve data integrity.

  • customers: Stores primary customer information. Links to the `plans` table via a plan_id foreign key.
  • plans: Contains all available billing plans, their names, and monthly charges. This allows plan details to be updated in one place.
  • invoices: Tracks billing for each customer, linked via customer_id. Includes due dates and payment status.
  • payments: Records every payment transaction against a specific invoice, linked by invoice_id.

This normalized structure ensures that data is stored logically and efficiently, preventing anomalies during data insertion, updates, or deletion.
