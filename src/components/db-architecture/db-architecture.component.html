Database Architecture

A conceptual model of the ISP Billing schema, demonstrating 3NF normalization and key relationships for a scalable system.

customers

id (PK)
name VARCHAR(255)
email VARCHAR(255)
plan_id (FK) - Links to plans table
join_date DATE

plans

id (PK)
plan_name VARCHAR(100)
speed_mbps INTEGER
monthly_charge DECIMAL(10,2)

invoices

id (PK)
customer_id (FK) - Links to customer record
invoice_date DATE
due_date DATE
total_amount DECIMAL(10,2)
status ENUM('Paid', 'Unpaid', 'Overdue')

payments

id (PK)
invoice_id (FK) - Links to paid invoice
amount DECIMAL(10,2)
payment_date DATETIME
method VARCHAR(50)

Performance & Indexing

To ensure fast query performance, indexes should be created on foreign keys (`plan_id`, `customer_id`, `invoice_id`) and frequently queried columns like `customers.email` and `invoices.status`. This minimizes table scan time and is crucial for a responsive application at scale.
